使用方法

这是一个Java命令行文件。阁下可使用cmd,git bash等命令行工具执行。

  • 开始执行时，命令行会提示Input sockets:；此时，在命令行内，**一行输入一个端口名称，并在输入完成后输入“-1”表示输入完成。

  • 端口输入完成后，命令行将提示Input instruction:；此时，请输入要加入的命令名称；

  • 输入命令名称后，命令行将提示Input signal:；此时，在命令行内，输入一个信号名称。输入后，命令行将提示Input destination port:；此时，在命令行内，输入其中一个先前已经输入的端口，表示输入的信号将接入输出的端口。

    **注意：

    1. 数据通路表中可能存在任意信号“XX”，此处我们用“/”表示。在输入“/”作为信号时，程序会提示” “/“ detected. Not adding it to the list. “，并且最后也不会将该信号加入表内。
    2. 若此时输入的端口在前一步中并未输入，那么在实际执行时，会提示”No specified port found. Signal input invalid.”，且本次输入的端口与上次输入的信号无效。

  • 输入完成后，会再次回到Input signal；此时可继续添加，或输入“-1”完成。

  • 随后，会回到Input instruction:。此时，输入”-1”，将结束整个输入过程。

  • 输入过程完成后，程序将提示选择操作数。

  • 操作1：输入一个接口，程序将返回接入此接口的所有信号。

  • 操作2：输入一个接口，随后在输入一个信号，程序将返回这一信号由那几条指令调用。

  • 操作3：输出统计数据，将输出每个端口接入的所有信号，以及各信号来自哪些函数。

  • 操作0：退出程序。

在包内，小弟附上了一份示例输入in.txt，便于重定向输入使用，同时也给出（无比丑陋的）输出out.txt。

背景

在我们进行工程设计的时候，我们需要对数据通路表进行查询，从而进行是否插入MUX/MUX接入什么控制信号/单元接入什么控制信号的判断。

在后续工程量由支持个位数指令至约40条指令时，查数据通路表的难度将会飞升，为开发带来极大的挑战。

为此，若有一数据通路表查询机，将会对我们的开发带来极大的便利。

功能要求

为了后续流水线拓展，我们需要把单周期数据通路表中，指令和输入接口的行列顺序换一下！

对于一行（一个输入端口），给出所有可能、且不重复的输入信号，用于判断是否换MUX/MUX接入的信号

同样对于一行，给出一输入信号，返回所有输入此输入信号的指令（由行找列），用于构建MUX的控制信号

约定

对于模块与MUX，我们采用全大写，无下划线的命名

对于控制信号，我们选用全小写，加下划线的命名

对于输入信号，遵循<module>.socket(@<level>)的命名。

思路

先造一容器存储所有模块的所有接口，再对各个接口造一个容器，其中按输入信号存储执行该信号的指令。

HashMap<String,HashMap<String,ArrayList<String>>>，对应<socket,<signal,instr>>

首先，我们将所有接口输入。

随后，输入一条指令的数据通路。先输指令名，再依次输入各接口接入的信号，格式为<signal> -> <socket>。

在输入完成后，我们便可进行查询。需要指示查询方法。

0 - 查询一个接口接入的所有信号

1 - 对一个接口，查询一个信号对应的所有指令

第一次尝试

万事开头难。

有关建设个人博客这件事，本人其实早在2022年5月，就已按各路教程，用Hexo和Github Pages搭好了环境。苦于当时的学业压力，加上后续也没有任何写随笔的时间和欲望，这一项目就此搁浅，主页上也只留着默认的“hello world”。

转眼，过了一年半，这个想法又被提上了日程。主要的推动力，我想是高宁老师在几周前《大国领袖》课程上的点醒。的确，写作使人深刻。思想在矛盾中产生进步，而若不以文字记录下所思所想，任何进步所获，都将被时间冲淡，最后终将导致自己寸步未进。听得此话，如醍醐灌顶。也许，真的是时候认真记录一下自己的所思所想了。

初入大学数月，感受颇丰——无论是学业上，还是这几个月来自己学余的所见所闻。这几个月，有看完MyGo!!!!!、补完少歌之后的感动，有学业繁重、认识到自己与各路大神差距的压力，有对自己选择信息类，而非三年前自己所热爱的航空学的怀疑，更有初中以后，自己的精神支柱长期空缺的反思。上述种种，我想，绝非只是自己心里想想，然后任其随时间逝去就能过去的。用文字记录下自己的感受，并在写作中反思，我想才是正解。

当然，自己搭建Blog，也绝非易事。在您观看这篇文章时，本人可能还在潜心研究NexT主题的各类复杂设置，这个博客也大概率还只是一个刚搭好的毛胚房。这也算是我第一次真正使用Github开发自己的项目，要学习的东西自然也是很多。

写到这里，已然词穷。本人Blog的第一篇文章，就到此草草作结吧。

但愿写完这篇文章之后，我还能记得继续写吧（笑
